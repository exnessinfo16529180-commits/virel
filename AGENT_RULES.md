# Agent Rules — Параллельная разработка Claude + Codex

Правила для безопасной параллельной работы в проекте Virel.

---

## Разделение ответственности

### Claude Code (этот агент)
**Владеет:**
- Архитектурные решения и core flow
- State management и бизнес-логика
- Роутинг и navigation structure
- Интеграции с API (Stitch и другие)
- Компонентная иерархия и data flow
- Финальная интеграция PR от других агентов
- Code review всех входящих изменений

**Файлы под прямым контролем:**
- `src/store/**` — state management
- `src/lib/**` — утилиты и integrations
- `src/router/**` — роутинг
- `src/App.tsx`, `src/main.tsx` — точки входа

---

### Codex (OpenAI)
**Владеет:**
- Изолированные UI-компоненты (атомарные)
- Визуальный refactor в ограниченном скоупе
- Инфраструктурные задачи (конфиги, scripts)
- Документация и комментарии
- Bugfix в явно очерченных файлах
- Стилизация и анимации

**Файлы под контролем Codex:**
- `src/components/ui/**` — базовые UI-блоки
- `src/styles/**` — CSS/стили
- `docs/**` — документация
- Конкретные файлы, указанные в TASK_CARD

---

## Правила параллельной работы

### Принципы
1. **Один агент = одна ветка = один PR** — без исключений
2. **Запрещено** одновременно менять одни и те же файлы разными агентами
3. **Все изменения** только через PR, никаких прямых пушей в `main`
4. **Merge в main** только после: ✅ green build + ✅ preview review founder'ом
5. **Конфликты** разрешает Claude как интегратор

### Соглашение по именованию веток

| Агент | Формат | Пример |
|-------|--------|--------|
| Claude | `claude/<feature>` | `claude/auth-flow` |
| Codex | `codex/<task>` | `codex/button-component` |
| Founder | `fix/<issue>` | `fix/typo-onboarding` |

### Перед открытием задачи Codex — Claude проверяет:
- [ ] Нет открытых PR от Codex с пересекающимися файлами
- [ ] TASK_CARD заполнена по шаблону из `TASK_CARD_TEMPLATE.md`
- [ ] Scope файлов явно ограничен
- [ ] Acceptance criteria измеримы

### Запрещённые действия для любого агента
- ❌ Прямой push в `main`
- ❌ Force push в любую ветку
- ❌ Изменение `.github/workflows/` без явного разрешения
- ❌ Коммит `.env` файлов
- ❌ Merge без прохождения CI build
- ❌ Изменение файлов вне своего Scope без согласования

---

## Workflow интеграции

```
Claude формирует TASK_CARD
        ↓
Codex создаёт ветку codex/<task>
        ↓
Codex открывает PR с описанием
        ↓
CI Build запускается автоматически
        ↓
Claude делает code review
        ↓
Founder approves с телефона (preview ok)
        ↓
Merge → Production deploy
```

---

## Разрешение конфликтов

Если два агента случайно трогают одни файлы:
1. Последний PR ждёт (остаётся в Draft)
2. Первый PR мержится (если зелёный)
3. Второй PR rebases от main
4. Claude проверяет конфликты перед merge
